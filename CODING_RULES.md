# Coding Rules

-   Never rewrite the whole file if only one engine changes.
-   Always preserve compileability.
-   One commit = one feature.
-   Never remove existing inputs unless replacing them.
-   Never repaint.
-   Always prefer performance over visual effects.
-   If uncertain, preserve existing behavior.

## Before Every Commit

-   Compile mentally.
-   Check variable names.
-   Check dashboard.
-   Check alerts.
-   Check Pine Script v6 compatibility.
